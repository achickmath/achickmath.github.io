Technical Projects

OmniBlade - Adaptive Motion Controller

Discovery Project | March 2026

Overview

OmniBlade is an adaptive game controller that uses arm motion instead of traditional buttons and joysticks. Built for players with limb differences, tremors, or limited hand mobility, it turns natural arm gestures into game inputs. Tilt your arm to move, flick your wrist to attack. No buttons, no setup, just motion.

The Problem

Accessibility controllers exist, but they all assume you can use your fingers. The Xbox Adaptive Controller remaps buttons to bigger buttons. Eye trackers need cameras and calibration. Switch access devices require mounting hardware and more setup time than actual gameplay. Every solution adds complexity while still requiring fine motor control in the hands.

For players with limb differences or limited hand mobility, the problem was never which button to press — it's that buttons were never designed for them in the first place. OmniBlade eliminates buttons entirely. If you have an arm, you can play.

Technical Implementation

The system uses two ESP32 DEVKIT V1 microcontrollers, each paired with an MPU-6050 6-axis IMU, mounted in custom 3D-printed forearm housings with velcro straps — one per arm. Each ESP32 reads sensor data over I2C at 50Hz and runs a complementary filter to fuse accelerometer and gyroscope readings into stable roll and pitch angles.

The complementary filter balances both sensors — trusting the gyroscope for fast motion and the accelerometer for slow corrections. This prevents the drift you'd get from using just the gyro and the noise you'd get from just the accelerometer. Flick gestures are detected by monitoring gyroscope spike thresholds with cooldown logic to prevent false triggers.

Both ESP32s stream data over USB serial to a Python host application. A multithreaded script reads both serial ports simultaneously and maps left arm tilt to the left analog stick and right arm tilt to the right analog stick via vgamepad, a virtual Xbox controller library. Flick gestures map to face buttons and triggers. Voice commands via VoiceAttack handle supplemental inputs including start, select, d-pad, and stick clicks — fully replacing all remaining controller inputs without any physical buttons.

The result is complete Xbox controller emulation driven entirely by arm motion and voice. Because it emulates a standard controller, it works out of the box with any game without remapping or custom profiles.

Results

OmniBlade was demonstrated across Elden Ring, Forza Horizon 5, and Devil May Cry 5 — including a live boss kill in Elden Ring using only motion controls. A separate analog trigger profile was developed for Forza, mapping right arm tilt forward to throttle and back to brake for proportional acceleration and braking. The project won Best in Thread for Embedded Devices at Georgia Tech's Innovate ECE Thread Challenge.

The biggest technical challenge was per-arm calibration and threshold tuning. Each arm has a different natural range of motion, requiring independent deadzone and max angle values for left and right stick normalization. Getting stable orientation from noisy IMU data while keeping latency low enough for real-time gameplay required extensive iteration.

What I Learned

This project taught me I2C communication and sensor fusion on embedded hardware, complementary filtering for IMU orientation estimation, multithreaded serial parsing in Python, and virtual input simulation on Windows and Linux via vgamepad. I also learned how to design gesture vocabularies around physical constraints — mapping a full Xbox controller to eight flick directions, two analog tilt axes, and voice forced hard decisions about which inputs actually matter and which could be deprioritized.

Future Development

The current build is a working prototype but not yet a product. Next steps include implementing a Madgwick filter for better orientation accuracy under fast motion, upgrading to an IMU with a magnetometer for stable yaw, and eventually designing a custom PCB to miniaturize the system into a wrist-mounted form factor closer to a wearable device. The goal is a controller that is as easy to put on as a watch and requires no setup — expressive enough to handle any game, for any player.

Tech Stack: ESP32, MPU-6050 IMU, Python, USB Serial, Complementary Filter, vgamepad, pynput, VoiceAttack, Multithreaded Serial Parsing

LTC2308 ADC Peripheral

FPGA Design Project | March 2026 - April 2026

Designed a memory-mapped VHDL peripheral integrating the LTC2308 ADC into a 16-bit processor I/O bus over SPI, supporting 8 selectable analog input channels with full clock generation, bit serialization, and chip select timing. Architected a 2-register MMIO interface abstracting all SPI protocol complexity from software, mirroring real-world hardware peripheral driver design. Implemented on-demand conversion triggering with a packed status-and-result read register, enabling efficient single-instruction polling. Validated SPI timing integrity and analog data accuracy across all channels using ModelSim simulation, FPGA hardware testing, and logic analyzer verification.

Source: Analog Devices
